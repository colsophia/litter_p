# 一：了解web及网络基础

# 1、网络基础TCP/IP

计算机和网络设备要相互通信，双方就必须基于相同的方法。那么如何探测到通信目标、由哪一边发起通信、使用哪种语言进行通信、怎样结束通信等规则都需要事先确定。不同的硬件、操作系统之间的通信就需要一种规则。我们就把这种规则称为协议（protocol）。

TCP/IP是互联网相关的各类协议族的总称。如TCP协议，IP协议，DNS协议，HTTP协议都是属于TCP/IP协议族的

TCP/IP的分层管理：

分层的好处：不用因为某些地方设计的变化，而去把所有的整体都替换掉，而只是需要把变动的层替换掉就可以

1、应用层：向用户提供应用服务时通信的活动（如：FTP文件传输协议，DNS域名解析协议，HTTP）

2、传输层：数据传输（如：TCP协议（Transmission Control Protocol 传输控制协议），UDP协议（User Date Protocol用户数据协议））

3、网络层：处理在网络上流动的数据包（数据包：网络传输的最小单位）。在众多的选项中选择一条传输路线（如路由器）

4、链路层：连接网络的硬件部分（如NIC（Network Interface Card 网络适配器）网卡，光纤）

# 2、IP、TCP、DNS、HTTP协议

2.1 IP协议：

1、负责传输

2、位于网络层

3、作用：将数据包传送给对方。为了保证确实传送到了对方那里，需要满足各种条件，其中最重要的2个：IP地址，MAX地址。

3.1 IP地址指明节点被分配到的地址，MAC地址是指网关所属的固定地址。

3.2 IP地址和MAC地址可以进行配对，IP地址可变换，但MAX地址基本不会改变。

1、ARP：（地址解析协议）Address Resolution Protocol

2、IP间的通信依赖于MAC地址。由于通信的双方经常需要经过中转才能通信（很少处于一个LAN中），在进行中转时，会利用下一站中转设备的MAC地址来搜索下一个中转目标

3、ARP是一种用于解析地址的协议，根据通信方的IP地址就可以反查出对应的MAC地址

2.2 TCP协议：

1、负责拆分、重组协议

2、位于传输层

3、作用：提供可靠的字节流服务（字节流服务：为了方便传输，将大块数据分割成以报文段为单位的数据包进行管理）

4、如何确保数据能到达目标：三次握手策略（three-way handshaking）

1、发送端发送一个带SYN标志的数据包给对方

2、接收方接收到数据发回一个带SYN/ACK标志的数据包给发送端，以示传送却达信息

3、发送端再传回一个带ACK标志的数据包，代表“握手”结束

2.3 DNS协议：

1、负责域名解析（提供域名到IP地址的解析服务）

2、位于应用层

3、作用：DNS协议提供通过域名查找IP地址，或逆向从IP地址反查域名的服务

2.4 HTTP协议：

1、负责针对目标web服务费生成请求报文，对web服务器的内容做处理

2、位于应用层

# 3、URI、URL

URI：统一资源标识符（Uniform Resource Identifier），用字符串标识某一互联网资源

URL：统一资源定位符（Uniform Resource Locator），标识资源的在互联网上所处的位置

URI格式：

http://user:pass@www.example.ip:80/dir/index.html?uid=1#ch1

http -- 协议方案名，后面需要加“:”符号

user:pass -- 登录信息（认证），可省略

@www.example.ip:80 -- 服务器地址：服务器端口号， 端口号可省略，不写为默认端口号

/dir/index.html -- 文件路径

uid=1 -- 查询字符串

ch1 -- 片段标识符，标记已获取资源中的子资源的位置
