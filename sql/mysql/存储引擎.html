# 二：存储引擎

mysql> show engines;
+--------------------+---------+----------------------------------------------------------------+--------------+------+------------+

+--------------------+---------+----------------------------------------------------------------+--------------+------+------------+
| MEMORY             | YES     | Hash based, stored in memory, useful for temporary tables      | NO           | NO   | NO         |
| MRG_MYISAM         | YES     | Collection of identical MyISAM tables                          | NO           | NO   | NO         |
| CSV                | YES     | CSV storage engine                                             | NO           | NO   | NO         |
| FEDERATED          | NO      | Federated MySQL storage engine                                 | NULL         | NULL | NULL       |
| PERFORMANCE_SCHEMA | YES     | Performance Schema                                             | NO           | NO   | NO         |
| MyISAM             | YES     | MyISAM storage engine                                          | NO           | NO   | NO         |
| InnoDB             | DEFAULT | Supports transactions, row-level locking, and foreign keys     | YES          | YES  | YES        |

从上图我们可以查看出 MySQL 当前默认的存储引擎是 InnoDB，并且在 5.7 版本所有的存储引擎中只有 InnoDB 是事务性存储引擎，也就是说只有 InnoDB 支持事务。

# 4、MyISAM 和 InnoDB 的区别

MySQL5.5之前，MyISAM引擎是MySQL的默认存储引擎；5.5版本后，MySQL引入了InnoDB,之后的版本都使用这个引擎。

存储引擎 是否支持行级锁 是否支持事务 是否支持外键
InnoDB 支持表级锁和行级锁。默认是行级锁 提供事务支持，具有提交和回滚事务的能力 支持外键
MyISAM 只支持表级锁 不提供事务支持 不支持外键

一般我们也是不建议在数据库层面使用外键的，应用层面可以解决。不过，这样会对数据的一致性造成威胁。具体要不要使用外键还是要根据你的项目来决定。

# 5、表级锁和行级锁

表级锁： MySQL中锁定粒度最大的一种锁，对当前整个表进行加锁，实现简单，消耗资源较少，加锁快，不会出现死锁。但是触发锁冲突的概率最高，并发度最低

行级锁： MySQL中锁定粒度最小的一种锁，默认只针对当前操作的行进行加锁。行级锁大大减少数据库操作的冲突。并发度高，但是加锁的开销较大，加锁慢，会出现死锁
