# 单例池、单例Bean、单例模式

Spring容器中存的是Bean

# 1、单例池、单例Bean、单例模式的区别

单例模式： 在一个jvm进程中，只能有一个实例

单例bean： 在一个Spring容器中，每个Bean只生成一个对象实例，每次getBean的时候都是获取的此实例，不会再new一个

单例池： 用户缓存单例bean

单例Bean != Spring容器里只能有一个这个类型的Bean。举例：

<?xml version="1.0" encoding="UTF-8"?>
<beans xmlns="http://www.springframework.org/schema/beans"
       xmlns:xsi="http://www.w3.org/2001/XMLSchema-instance"
       xmlns:context="http://www.springframework.org/schema/context"
       xsi:schemaLocation="http://www.springframework.org/schema/beans http://www.springframework.org/schema/beans/spring-beans.xsd http://www.springframework.org/schema/context https://www.springframework.org/schema/context/spring-context.xsd">
    <bean id="userInfo" class="com.example.springboot.domain.UserInfo"/>
    <bean id="userInfo1" class="com.example.springboot.domain.UserInfo"/>
</beans>

ClassPathXmlApplicationContext applicationContext = new ClassPathXmlApplicationContext("spring.xml");
UserInfo userInfo = applicationContext.getBean("userInfo", UserInfo.class);
UserInfo userInfo1 = applicationContext.getBean("userInfo1", UserInfo.class);
// 结果是两个不同的bean    beanName = userInfo 的为单例bean；  beanName = userInfo1 的也是单例bean；

# 2、单例池的结构和作用

<?xml version="1.0" encoding="UTF-8"?>
<beans xmlns="http://www.springframework.org/schema/beans"
       xmlns:xsi="http://www.w3.org/2001/XMLSchema-instance"
       xmlns:context="http://www.springframework.org/schema/context"
       xsi:schemaLocation="http://www.springframework.org/schema/beans http://www.springframework.org/schema/beans/spring-beans.xsd http://www.springframework.org/schema/context https://www.springframework.org/schema/context/spring-context.xsd">
    <bean id="userInfo" class="com.example.springboot.domain.UserInfo"/>
</beans>

ClassPathXmlApplicationContext applicationContext = new ClassPathXmlApplicationContext("spring.xml");
UserInfo userInfo = applicationContext.getBean("userInfo", UserInfo.class);
UserInfo userInfo1 = applicationContext.getBean("userInfo", UserInfo.class);
//  userInfo和userInfo1是一个对象

一个单例bean，每次getBean的时候都是获取到的同一个对象。这就是单例池的作用，起到一个缓存的效果。会把bean保存到ConcurrentHashMap中，key就是beanName，value就是这个bean对象
