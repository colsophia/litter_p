# 九：ApplicationContext是什么

ApplicationContext是比BeanFactory更强大的Spring容器，它即可以创建bean,获取bean,还支持国际化、事件传播、获取资源等BeanFactory不具备的功能

# 1、ApplicationContext所继承的接口

public interface ApplicationContext extends EnvironmentCapable, ListableBeanFactory, HierarchicalBeanFactory, MessageSource, ApplicationEventPublisher, ResourcePatternResolver {}
  • EnvironmentCapable： 可以通过ApplicationContext获取操作系统环境变量和JVM环境变量。
  • ListableBeanFactory： ListableBeanFactory继承自BeanFactory。可以获取所有BeanDefinition对象，统计BeanDefinition的个数，获取某个类型对应的所有beanNames等功能
  • HierarchicalBeanFactory： 获取父BeanFactory，判断某个name是否存在bean对象的功能
public interface HierarchicalBeanFactory extends BeanFactory {
    @Nullable
    BeanFactory getParentBeanFactory();

    boolean containsLocalBean(String var1);
}
  • MessageSource： 拥有国际化功能。如不通国家语言所对应的字符
  • ApplicationEventPublisher： 拥有事件发布功能，可以发布事件。这是ApplicationContext相对于BeanFactory比较突出，常用的功能。
@FunctionalInterface
public interface ApplicationEventPublisher {
    default void publishEvent(ApplicationEvent event) {
        this.publishEvent((Object)event);
    }

    void publishEvent(Object event);
}
  • ResourcePatternResolver： 拥有加载并获取资源的功能，资源可以是文件，图片等某个URL资源都可以
public interface ResourcePatternResolver extends ResourceLoader {
    String CLASSPATH_ALL_URL_PREFIX = "classpath*:";

    Resource[] getResources(String locationPattern) throws IOException;
}
