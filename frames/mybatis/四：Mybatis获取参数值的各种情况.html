# 四：Mybatis获取参数值的各种情况

# 1、mapper接口方法的参数为单个的字面量类型

# 总结：

通过#{}和${}以任意的名称获取参数值，但是需要注意${}的单引号问题

# 1.1 采用 #{ }

<!-- User getUserByName(String username); -->
<select id="getUserByName" resultType="User">
      SELECT * FROM user WHERE username = #{username}
</select>

DEBUG 05-02 12:10:27,994 ==>  Preparing: SELECT * FROM user WHERE username = ? (BaseJdbcLogger.java:137) 
DEBUG 05-02 12:10:28,027 ==> Parameters: admin(String) (BaseJdbcLogger.java:137) 
DEBUG 05-02 12:10:28,043 <==      Total: 1 (BaseJdbcLogger.java:137)

‘#’号是采用问号占位，然后进行替换，至于 xml中的 #{参数名}，参数为单个的字面量类型，#{}中的参数名可以随意命名，至于位置有关，与名字无关，一般来说都取名一样，便于理解

# 1.2 采用${ }

<!-- User getUserByName(String username); -->
<select id="getUserByName" resultType="User">
      SELECT * FROM user WHERE username = ${username}
</select>

DEBUG 05-02 16:42:03,619 ==>  Preparing: SELECT * FROM user WHERE username = admin (BaseJdbcLogger.java:137) 
DEBUG 05-02 16:42:03,668 ==> Parameters:  (BaseJdbcLogger.java:137)

‘$’号是采用字符串拼接，不能直接采用上述办法，需要手动加上单引号，如下

<!-- User getUserByName(String username); -->
<select id="getUserByName" resultType="User">
      SELECT * FROM user WHERE username = '${username}'
</select>

DEBUG 05-02 16:43:41,479 ==>  Preparing: SELECT * FROM user WHERE username = 'admin' (BaseJdbcLogger.java:137) 
DEBUG 05-02 16:43:41,515 ==> Parameters:  (BaseJdbcLogger.java:137) 
DEBUG 05-02 16:43:41,539 <==      Total: 1 (BaseJdbcLogger.java:137)

# 2、mapper接口方法的参数为多个时

此时Mybatis会将这些参数放在一个map集合中，以2中方式存储：

  • 以arg0,arg1...... 为键，以参数为值
  • 以param1,param2...... 为键，以参数为值

因此只需要通过#{}和${}以键的方式访问值即可，但是需要注意${}的单引号问题

# 3、若mapper接口方法的参数有多个时，可以手动将这些参数放在一个map中存储

只需要通过#{}和${}以键的方式访问值即可，但是需要注意${}的单引号问题

<!-- User checkUserByLogin(Map<String,String> map); -->
<select id="checkUserByLogin" resultType="User">
    SELECT * FROM user WHERE username = #{username} AND password = #{password}
</select>

@Test
public void testUserLogin() throws IOException {
   SqlSession sqlSession = SqlSessionUtils.getSqlSession();

   ParameterMapper mapper = sqlSession.getMapper(ParameterMapper.class);
   Map<String, String> map = new HashMap<>();
   map.put( "username", "admin");
   map.put( "password", "1234");
   User admin = mapper.checkUserByLogin(map);
   System.out.println(admin);
}

# 4、mapper接口方法的参数是实体类类型的参数

只需要通过#{}和${}以属性的方式访问属性值即可，但是需要注意${}的单引号问题

<!-- int addUser(User user); -->
<insert id="addUser">
   INSERT INTO user(id,username,password,sex,email) VALUE(null, #{username}, #{password}, #{sex}, #{email})
</insert>

@Test
public void testAddUser() throws IOException {
  SqlSession sqlSession = SqlSessionUtils.getSqlSession();

  ParameterMapper mapper = sqlSession.getMapper(ParameterMapper.class);
  int row = mapper.addUser(new User("sophia","1231456","女","1044545111@qq.com"));
  System.out.println(row);
}

DEBUG 05-02 17:33:32,090 ==> Parameters: sophia(String), 1231456(String), 女(String), 1044545111@qq.com(String) (BaseJdbcLogger.java:137)

# 5、使用@Param命名参数

此时mybatis会将这些参数放在一个map集合中，以两种方式存储

  • 以@Param注解的值为键，以参数为值
  • 以param1,param2...... 为键，以参数为值

因此只需要通过#{}和${}以键的方式访问值即可，但是需要注意${}的单引号问题

<!-- User checkUserByLogin2(@Param("username") String username, @Param("password") String password);  -->
<select id="checkUserByLogin2" resultType="User">
    SELECT * FROM user WHERE username = #{username} AND password = #{password}
</select>
