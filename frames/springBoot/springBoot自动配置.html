# 二：自动配置

# 1、自动配置好Tomcat

1、引入Tomcat依赖（比如，我们引入了spring-boot-starter-web，就会自动帮我们引入Tomcat场景）

2、配置Tomcat

<dependency>
   <groupId>org.springframework.boot</groupId>
      <artifactId>spring-boot-starter-tomcat</artifactId>
      <version>2.6.3</version>
      <scope>compile</scope>
</dependency>

# 3、自动配好Web常见功能

如：字符编码问题(characterEncodingFilter)

SpringBoot帮我们配置好了所有web开发的常见场景

@SpringBootApplication
public class SpringbootDemoApplication {

    public static void main(String[] args) {
        ConfigurableApplicationContext run = SpringApplication.run(SpringbootDemoApplication.class, args);
        // 获取所有组件定义的名字
        String[] names = run.getBeanDefinitionNames();
        for (String name: names) {
            System.out.println(name);
        }
    }
}

# 4、默认的包结构

Developing with Spring Boot (opens new window)

1、主程序（由@SpringBootApplication修饰的类）所在包及其下面的所有子包里面的组件都会被扫描进来

2、无需以前的包扫描配置

3、想要改变扫描路径，就可以使用@SpringBootApplication(scanBasePackages = "包路径")或者加上@ComponentScan("包路径")

@SpringBootApplication  注解等同于下面三个注解：
    @SpringBootConfiguration
    @EnableAutoConfiguration
    @ComponentScan("包路径")

# 5、各种配置拥有默认值

1、比如tomcat配置有默认端口，springmvc配置的东西也都有默认值。

2、默认配置的值最终都是映射到某个类上的，这个类会在容器中创建对象

如：配置server.port = 8099，则是映射到ServerProperties类上

# 6、按需加载所有自动配置项

1、有非常多是starter

2、引入了哪个场景，这个场景的自动配置才会开启

3、Springboot的自动配置功能都在spring-boot-starter 这个jar包中的spring-boot-autoconfigure里面

如：

1.1 现在我们的pom文件，只引入了spring-boot-starter-web这么一个jar包：

<dependencies>
    <dependency>
        <groupId>org.springframework.boot</groupId>
        <artifactId>spring-boot-starter-web</artifactId>
    </dependency>
</dependencies>

1.2 因为我们springboot项目的父项目spring-boot-starter-parent的父项目spring-boot-dependencies里面有很多的starter场景，其中有一个就是spring-boot-autoconfigure（springboot自动配置场景）。我们在maven中找到这个jar包，并随意打开一个我们没有引入的场景aop代理，如下图所示，有爆红，简单的说就是没有开启自动配置。

1.3 我们在pom文件中引入aop的场景，并刷新我们maven，后看到，刚刚那个爆红的文件就没有在爆红了，说明我们引入了这个aop的场景，才会开启aop的自动配置。具体原因后续再讲解

<dependencies>
  <dependency>
     <groupId>org.springframework.boot</groupId>
      <artifactId>spring-boot-starter-web</artifactId>
   </dependency>
   <dependency>
       <groupId>org.springframework.boot</groupId>
       <artifactId>spring-boot-starter-aop</artifactId>
   </dependency>
</dependencies>

# 7、总结

所谓的自动配置：其实就是以spring-boot-starter-开头的jar包都会依赖于一个spring-boot-starter的基础jar包，而这个spring-boot-starter里面 就引入了一个spring-boot-autoconfigure依赖（自动配置）；只有在pom文件中引入了的jar包，才会开启自动配置
