# 一：依赖管理

# 1、父项目的依赖管理

<!-- springboot项目都会将这个jar包作为父项目 -->
<parent>
    <groupId>org.springframework.boot</groupId>
    <artifactId>spring-boot-starter-parent</artifactId>
    <version>2.6.3</version>
    <relativePath/>
 </parent>

<!-- 下面是spring-boot-starter-parent的父项目 -->
<parent>
    <groupId>org.springframework.boot</groupId>
    <artifactId>spring-boot-dependencies</artifactId>
    <version>2.6.3</version>
</parent>

spring-boot-dependencies中在properties中声明了很多常用的依赖的版本号。部分代码如下：
<properties>
     <h2.version>1.4.200</h2.version>
     <httpclient.version>4.5.13</httpclient.version>
     <mysql.version>8.0.28</mysql.version>
     <tomcat.version>9.0.56</tomcat.version>
     <thymeleaf.version>3.0.14.RELEASE</thymeleaf.version>
</properties>

# 2、开发导入stater场景启动器

1、见到很多 spring-boot-starter-*，这些都是官方提供发
2、只要引入starter，这个场景的所有常规需要的依赖都自动引入
3、Springboot所有支持的场景：
https://docs.spring.io/spring-boot/docs/current/reference/html/using.html#using.build-systems.starters
4、见到的*.spring-boot-starter，是第三方提供的简化的场景启动器 
5、所有的场景启动器，最基本（最底层）的依赖就是spring-boot-starter
	<dependency>
          <groupId>org.springframework.boot</groupId>
          <artifactId>spring-boot-starter</artifactId>
          <version>2.6.3</version>
          <scope>compile</scope>
	</dependency>

1、上图，我们引入了 spring-boot-starter-web的场景，他自动的就帮我们引入了tomact的场景，
        springmvc场景，sping的web开发模块，json开发的场景
2、所有的场景启动器，最基本的依赖就是spring-boot-starter

# 3、无需关注版本号，自动版本仲裁

引入spring-boot-dependencies里面申明了的依赖，就不需要写版本号。
但是为了方便版本号的管理，通常都统一写在properties中

# 4、可以修改版本号

maven仓里：https://mvnrepository.com/

1、查看spring-boot-dependencies里面规定当前依赖的版本，用的 key 
2、在当前项目里面重新配置
<properties>
    <mysql.version>5.1.38</mysql.version>
</properties>

# 5、总结

所谓的依赖管理，其实就是springboot帮我们封装好了很多常用的一些依赖，引入一个场景器就会自动的帮我们管理一下依赖

比如引入了spring-boot-starter-web，就会自动的帮我们引入tomcat，mvc，json，web依赖项
